serialize

  • serialize(): void

  • Apply pending changes into the packet and serialize it into a new internal buffer (the one that getBuffer will later return).

    Returns void

    Remarks

    NOTE: In most cases there is no need to use this method. It must be called only if the application retrieves information from the packet (by calling getBuffer, getPayload, getExtension, etc) and modifies the obtained buffers in place. However, it's recommended to use the existing setter methods instead (setPayload, setExtension, etc).

    Throws

    if invalid fields were previously added to the packet.

    • Defined in rtp.ts:734
